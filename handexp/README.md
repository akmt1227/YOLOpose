# 手キーポイント実験（MediaPipe Hands）

俯瞰カメラ映像で**指レベルのジェスチャー検出**が可能かの実現可能性検証。
本体（oneclass/・教師あり版）からは完全に独立。

## 結論: 実現可能 ✅

**固定ゾーンクロップ ＋ MediaPipe HandLandmarker ＋ 指ごとの伸展判別**で、
白手袋・俯瞰視点でも「指差しジェスチャーそのもの」を検出できる。

### 検証で分かった3つの事実

1. **白手袋は致命傷ではない**: 手がはっきり見えるフレームでは21点が正確にフィット
2. **手首キーポイント基準のクロップは失敗する**: 指差し中は手首の信頼度が崩壊し
   クロップ位置がずれる → **作業エリアの固定ゾーン**（固定カメラの利点）を常時
   クロップする方式が正解
3. **人差し指伸展だけでは不十分**: 開いた平手でも高値になる →
   **「人差し指が伸び(idx>1.6) かつ 他の指が曲がる(others<1.3)」**で指差しを判別

### 検証結果（指差しなしNG動画）

| 区間 | 指差し判定フラグ |
|---|---|
| 32〜34秒（真の指差し1） | 32.9秒×2・34.7秒×2 ✅ |
| 44〜46秒（真の指差し2） | 44.7/44.8/45.3/45.7秒 ✅ |
| 22〜26秒（トレイ動作） | 23.0秒に単発1回のみ → 「1.5秒内に2回以上」の集約で除去可能 |

## ファイル

- `hand_landmarker.task` — MediaPipeモデル（自動DL済み、gitignore対象外の7.8MB）
- `feasibility.py` — 初期検証（全画面 vs 手首クロップ）
- `zone_test.py` — 固定ゾーン方式の検証＋検出率測定
- `finger_metric.py` — 指ごとの伸展判別式の検証
- `out/` — 注釈付き検証画像

## 検出器の設計（次ステップで実装する場合）

```
毎フレーム（or 2フレームに1回・CPU動作なのでYOLOと並列可）:
  作業ゾーン(455,440,560px)をクロップ → 2倍拡大 → HandLandmarker
  → 各手について idx>1.6 and max(mid,rng,pnk)<1.3 なら「指差し姿勢」
イベント化: 1.5秒以内に2回以上の指差し姿勢 → 指差し確認1回と認定
```

- ゲート4の「本物の指差し検出器」として組み込めば、プロキシ/厳密検出の
  複合構成を置き換えられる見込み
- 他工程への展開: ゾーン座標と指ルールを差し替えるだけで「指差し」「つまみ」
  「持ち替え」等のジェスチャー検出に流用可能

## 制約・注意

- 検出率はフレーム単位で17〜37%（手の向き・遮蔽に依存）→ **瞬間の検出は
  不可、1〜2秒のイベント集約が前提**
- ゾーン座標はカメラ・工程ごとに較正が必要
- mediapipe==0.10.35 をvenvに追加済み（requirements.txt には未追加・実験扱い）
