# ワンクラス異常検知 試作（One-Class Prototype）

**良品（正常作業）の動画だけを学習**し、そこから外れるもの（見慣れない動き・作業していない状態）にNGを発報する試作です。NG動画のラベル整備が不要で、**未知のNGパターン**も「正常から外れた」として検知できます。

> このフォルダは本体（教師あり5クラス版）から**完全に独立**しています。本体のファイルは一切変更・参照していません（特徴量コードは同一仕様の独立コピー）。生成物（.npy / .pth / threshold.json）もこのフォルダ内に保存されます。

## 仕組み（3つのゲート）

| ゲート | 方式 | 発報内容 |
|---|---|---|
| 1. 逸脱検知 | **LSTMオートエンコーダ**。正常作業の13秒窓だけで「復元」を学習。復元誤差が閾値超え＝見慣れない動き | `NG: DEVIATES FROM NORMAL WORK` |
| 2. 静止検知 | ルール。四肢の動きエネルギーが正常レンジを大きく下回る窓 | `NG: WORKER IDLE` |
| 3. 不在検知 | ルール。作業者が10秒以上検出されない | `NG: NO WORKER PRESENT` |
| 4. 指差し確認ウォッチドッグ | ルール。検査ボード方向へ腕を伸ばす指差しジェスチャー（作業者の右手。背面カメラのためYOLOラベル上は"左手首"）が**初回16秒／以降20秒**観測されない。バナーに経過秒数を表示 | `NG: NO POINTING CHECK (20s)` |

- 異常スコア＝窓内の**タイムステップ別復元誤差の上位20%平均**（短い異常が平均に埋もれないため）
- 閾値は**正常データだけで自動較正**（ホールドアウトした正常動画のスコア分布の99パーセンタイル）
- 出力動画の右上に**リアルタイムのスコア/閾値**を表示（チューニング用）

## 使い方

### GUI（推奨）

```bash
uv run streamlit run oneclass/app.py
```

サイドバーの3ステップ（抽出→学習→検知）を順に実行するだけです。
- パイプラインの進行状態をバッジ表示（前段が未完了ならボタンは無効化）
- 実行ログをリアルタイム表示
- 学習後は較正されたしきい値をメトリクス表示、検知時はGUIから手動調整も可能
- 検知後はNG発報の一覧・フレームプレビュー・動画再生

### CLI（リポジトリのルートから実行）

```bash
# 1) 正常動画から窓を抽出（dataset/normal/*.mp4 を使用）
uv run python oneclass/prepare_normal.py

# 2) オートエンコーダ学習＋閾値較正（正常のみ・NG動画不要）
uv run python oneclass/train_ae.py

# 3) 検知テスト
uv run python oneclass/detect.py --input path/to/video.mp4 --output result_oneclass.mp4
```

### 調整オプション

| やりたいこと | 方法 |
|---|---|
| 感度を上げる（誤報増を許容） | `train_ae.py --percentile 95`（既定99）、または `detect.py --threshold 0.01` で直接指定 |
| 正常動画の場所を変える | `prepare_normal.py --normal_dir <path>` |
| YOLOモデルを変える | `pose_features.py` の `YOLO_WEIGHTS`（既定 `yolo26x-pose.pt`。初回に自動DL。CPU実行では重いので遅い場合は `yolo26m-pose.pt` 等へ） |
| サイクル長を変える | `pose_features.py` の `SEQ_LEN`（65=13秒@5fps） |
| 不在判定の秒数 | `detect.py` の `ABSENCE_SECONDS` |

## 制約（本体との使い分け）

- ゲート1（AE）は**NGの理由を特定できない**（「正常から外れた」のみ）。理由付きはルールゲート（2〜4）と本体の教師あり5クラス版の役割。
- ゲート1は**「抜け」系に構造的に弱い**（局所的にはすべて正常な動きの組合せのため）。指差し確認なしはゲート4のルールでカバー。他の「抜け」（外観検査なし等）は未カバー。
- ゲート4のジェスチャー判定パラメータは正常3本＋指差しなしNG1本でグリッドサーチ較正済み（`pose_features.py`）。**カメラ位置や指差し動作が変わったら再較正が必要**。
- 正常のバリエーション（作業者・ペースの違い）が学習データに無いと誤報しやすい。**多様な正常動画**を入れること。
- 将来的には本体と並走させるハイブリッド構成（既知NG=教師あり、未知NG=本試作）が推奨。

## ファイル

- `pose_features.py` — 姿勢特徴（55次元・13秒窓）＋動きエネルギー
- `model.py` — LSTMオートエンコーダ＋異常スコア計算
- `prepare_normal.py` — 正常動画→ `X_normal.npy` / `groups_normal.npy`
- `train_ae.py` — 学習＋較正 → `ae_model.pth` / `threshold.json`
- `detect.py` — 動画推論（3ゲート＋NGバナー＋スコア表示）
- `app.py` — Streamlit GUI（3ステップのワークフロー）
