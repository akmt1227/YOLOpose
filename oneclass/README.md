# ワンクラス異常検知 試作（One-Class Prototype）

**良品（正常作業）の動画だけを学習**し、そこから外れるもの（見慣れない動き・作業していない状態・指差し確認の欠落）にNGを発報する。NG動画のラベル整備なしで**未知のNGパターン**も検知できる。

> 本体（教師あり5クラス版）から完全に独立。生成物（.npy / .pth / threshold.json）もこのフォルダ内。

## 仕組み（4つのゲート）

| ゲート | 方式 | 発報内容 |
|---|---|---|
| 1. 逸脱検知 | **LSTMオートエンコーダ（6秒窓）**。正常作業の復元だけを学習し、復元誤差（上位20%ステップ平均・confチャンネル除外）が閾値超えで発報 | `NG: DEVIATES FROM NORMAL WORK` |
| 2. 静止検知 | ルール。四肢の動きエネルギーが正常レンジ未満 | `NG: WORKER IDLE` |
| 3. 不在検知 | ルール。作業者が10秒以上検出されない | `NG: NO WORKER PRESENT` |
| 4. 指差し確認 | **ハイブリッド**（下記） | `NG: NO POINTING CHECK (経過s)` |
| 5. 外観検査の実施 | **検査フェーズ・ウォッチドッグ**（下記） | `NG: SKIPPED INSPECTION (経過s)` |

### ゲート4: 指差し確認ウォッチドッグ（ハイブリッド構成）

指差しが**初回16秒／以降20秒**観測されないと発報。2つの検出器を実測特性に基づき使い分け:

- **発報防止（アーミング）**: 姿勢プロキシ（サイクル周期の腕伸ばし。正常動画での再現率が高い＝誤発報を防ぐ）
- **発報解除**: **指ベース検出器**（`hand_pointing.py`）— MediaPipe HandLandmarker（**VIDEOモード追跡**）を作業ゾーンの固定クロップに適用し、「人差し指伸展＋他指屈曲＋指先が指差し先エリア＋2.5秒内8ヒット」で**本物の指差しジェスチャーだけ**に反応（白手袋でも動作・検出率74%）。手の骨格は出力動画にオーバーレイ表示（指差し判定中は人差し指が赤）
- 保険: プロキシ2回連続でも解除（発報の張り付き防止）

### ゲート5: 外観検査ウォッチドッグ（OK動画のみで学習）

外観検査（体を捻り頭を下げ、ケーブルを回しながら約7秒目視）が**初回13秒／以降16秒**観測されないと発報。検査の判定は:

- 正常動画のみをk-means（16クラスタ）でフェーズ分解し、**検査クラスタ集合{5,6,11,15}**を較正（`train_inspection.py` → `inspection_phase.npz`）
- 補完条件: **C14（保持系姿勢）かつ低速（動き<0.024）** — 「ゆっくりケーブルを回す検査」と「テンポの速い通常作業」を動きで区別
- 2秒内3ヒットで「検査実施」。判定は**中心±1.1秒窓**で約1秒遅延評価（後ろ向き窓では遅い動作の開始で直前の速い動きが混入し検出が崩れることをE2Eで確認済み）

## 主要な設計判断（実験による裏付け）

- **窓長6秒**（2026-07-18採用）: 6/10/13秒比較で、検知マージン最良（1.22/1.55倍）・発報最速（初回判定6秒）・too_long/skipped感度最高・正常156秒で誤報ゼロ
- **stride 3秒維持**: 1秒化は一貫した効果なしと実測
- **単一ワーカー追跡**: トラッカーIDは激しい動作で断絶するため不使用
- **confチャンネルを復元誤差から除外**: 座り作業の遮蔽でconfがチラつき誤差の2/3を占めていた

## 最終検証結果（全ゲート統合・2026-07-18）

| 動画 | 発報 |
|---|---|
| drop（落下×2） | dev 5.8・19.6秒（**2/2**） |
| no_pointing（指差しなし×2） | 指差し監視16秒発報 → **32.9秒の本物の指差しを指検出して解除**（欠落2件を1発報でカバー） |
| too_long（検査時間過多） | dev×3 ＋ 指差し遅延発報16→17.6秒 |
| skipped（検査なし×2） | **`SKIPPED INSPECTION` 13秒発報 → 19.2秒の検査再開で解除**（欠落2件をカバー）＋ dev×2 |
| **正常コントロール156秒** | **発報ゼロ**（全5ゲート） |

## 使い方

### GUI（推奨）

```bash
uv run streamlit run oneclass/app.py
```

### CLI（リポジトリのルートから実行）

```bash
# 1) 正常動画から窓を抽出（dataset/normal/*.mp4）
uv run python oneclass/prepare_normal.py

# 2) オートエンコーダ学習＋閾値較正（正常のみ・NG動画不要）
uv run python oneclass/train_ae.py

# 3) 検知（バナー・OK/NGチップ・手骨格・スコアを描画した動画を出力）
uv run python oneclass/detect.py --input path/to/video.mp4 --output result.mp4
```

### 調整オプション

| やりたいこと | 方法 |
|---|---|
| 感度（AE） | `train_ae.py --percentile 95`（既定99）／`detect.py --threshold 値` |
| 窓長 | `pose_features.py` の `SEQ_LEN`（30=6秒@5fps）＋再抽出・再学習 |
| 指差し判定の較正 | `hand_pointing.py` のゾーン座標・指しきい値（カメラ／工程が変わったら要再較正） |
| 指差し期限 | `pose_features.py` の `POINTING_FIRST_DEADLINE_SECONDS` / `POINTING_TIMEOUT_SECONDS` |
| YOLOモデル | `pose_features.py` の `YOLO_WEIGHTS`（既定 yolo26x-pose、初回自動DL） |

## 制約

- ゲート1（AE）はNGの**理由を特定できない**（「正常から外れた」のみ）。「抜け」系の理由付き検知は本体の教師あり5クラス版の役割（指差しなしはゲート4がカバー）。
- 動画末尾で境界ブリップ（0.1秒程度の点滅発報）が出ることがある（実運用の連続映像では発生しない）。
- ゲート4のパラメータ（ゾーン・指先エリア・ヒット数）は本現場の映像で較正済み。**カメラ位置・工程が変わったら再較正が必要**。
- 依存: mediapipe（requirements.txt に追加済み）。手モデル `hand_landmarker.task` は初回に自動DL。

## ファイル

- `pose_features.py` — 姿勢特徴（55次元・6秒窓）＋プロキシ判定＋定数
- `model.py` — LSTMオートエンコーダ＋異常スコア計算
- `hand_pointing.py` — 指ベース指差し検出器（MediaPipe VIDEOモード）＋骨格描画
- `prepare_normal.py` — 正常動画→ `X_normal.npy` / `groups_normal.npy`
- `train_ae.py` — 学習＋較正 → `ae_model.pth` / `threshold.json`
- `train_inspection.py` — 検査フェーズのk-means学習 → `inspection_phase.npz`（ゲート5）
- `inspection_watch.py` — 検査ウォッチドッグ本体（クラスタ照合＋低速条件・中心窓評価）
- `detect.py` — 動画推論（4ゲート＋バナー＋チップ＋手骨格＋スコア表示）
- `app.py` — Streamlit GUI（3ステップのワークフロー）
